## 历史和规划文档

[功能需求与迭代规划](功能需求与迭代规划.md) 和工作区根目录 `doc/` 下的路线图用于记录阶段规划、实施进度和历史决策。它们可以帮助理解项目演进，但遇到具体操作步骤时，应以 README 快速开始、使用说明、API 文档和部署说明为准。

## 维护规则

- 新增或修改用户可见功能时，优先更新本索引中的权威文档。
- 只有当新文档有清晰读者、生命周期和导航入口时，才新增独立文档。
- 不把同一操作流程同时散落在多个文档中；专题文档可以保留细节，但应指向 [使用说明](使用说明.md) 中的端到端流程。
- 不在公开文档中写入真实服务器地址、密钥、连接串、备份包路径或本机私有部署脚本。
