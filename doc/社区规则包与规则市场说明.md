# 社区规则包与规则市场说明

本文档说明 LiteWaf 社区规则目录、远程规则包预览、信任密钥、更新审核和贡献导出流程。该能力只运行在控制面 API 和后台页面中，OpenResty 网关仍只读取已发布的可执行规则配置，不会在请求路径中拉取目录、校验签名或读取导出产物。

## 目录来源

社区目录来源可以是 HTTPS 地址或本地文件路径。目录创建只保存来源配置，不会自动拉取或启用规则；管理员必须显式点击同步。

目录文档格式：

```json
{
  "schema_version": "litewaf-rule-catalog-v1",
  "packages": [
    {
      "id": "litewaf-community-demo",
      "name": "LiteWaf Community Demo",
      "version": "v1.0.0",
      "compatibility": "litewaf-rule-package-v1",
      "checksum": "optional-sha256",
      "updated_at": "2026-06-01T00:00:00Z",
      "manifest_url": "https://example.invalid/litewaf-community-demo.json"
    }
  ]
}
```

目录项也可以直接内嵌 `package` 对象，适合离线镜像和本地验证。同步会保存包 ID、版本、兼容性、校验和、签名状态和来源身份；同步失败会记录错误并保留上一次成功同步的元数据。

## 远程包预览和更新

远程包预览复用本地规则包解析流程，返回包元数据、签名状态、兼容性、待新增规则、待变更规则、跳过规则和无效规则。目录同步和远程预览都不会创建、启用、禁用或发布规则。

更新检查按 `package_id + package_rule_id` 做确定性匹配。应用更新前，管理员应先查看新增、变更、移除、跳过和无效规则；应用更新只在明确确认后执行，且默认不启用包内声明为禁用的规则。

## 信任模型

信任密钥只保存和返回有界公开元数据，例如 key ID、算法、归属方、启用状态、撤销状态和过期时间。私钥、API Token、仓库凭证和签名材料不应进入 LiteWaf。

签名状态包括：

| 状态 | 含义 |
| --- | --- |
| `verified` | 签名校验通过且 key 可信 |
| `unsigned` | 包未携带签名 |
| `invalid` | 签名或 checksum 不匹配 |
| `untrusted-key` | key 不在信任库或被禁用 |
| `revoked-key` | key 已撤销 |
| `expired` | 签名或 key 已过期 |

本阶段未强制拒绝未签名包，但预览、导入结果、更新预览和发布预览都会显式展示风险。生产环境建议先在 `log-only` 或禁用状态下观察社区规则，再逐步调整动作。

## 导出贡献

导出流程分两步：

1. 导出预览：选择规则和包元数据，校验规则字段、重复包内规则 ID、兼容性和 checksum 计划。
2. 生成产物：确认后返回贡献包 JSON、checksum 和贡献提示。

导出产物不会包含私钥、API Token、Authorization/Cookie、原始流量样本、数据库连接串或部署密钥。若填写签名 key ID，LiteWaf 只写入签名元数据，实际私钥签名应在外部安全环境完成。

示例文件：

- `examples/community-catalog.json`
- `examples/community-rule-package.json`

## 网关边界

社区目录、信任库、更新记录和导出产物都属于控制面管理数据。发布时，网关配置只包含现有可执行规则结构和有限包来源元数据，例如 `package_id`、`package_version` 和 `package_rule_id`。目录不可用、信任库不可用或规则市场不可达，不会影响已发布配置的运行时拦截。
