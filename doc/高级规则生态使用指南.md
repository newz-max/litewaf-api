# 高级规则生态使用指南

English title: Advanced Rule Ecosystem Guide.

## English Guide

The advanced rule ecosystem connects local rule package preview, signature status, source tracking, and rule testing into a safe review loop. The first version supports local JSON rule packages only and does not automatically fetch remote rule sources.

Rule packages are JSON artifacts with package metadata, defaults, and rule entries. Preview validates compatibility, signatures, invalid rules, skipped rules, warnings, and source metadata without activating rules. Import creates or updates rules deterministically by `package_id + package_rule_id`; imported executable rules still require publishing before the Gateway enforces them.

Security boundary: signatures and trust state are control-plane metadata and publish-preview warnings. The Gateway enforces only published executable rules and never downloads packages, verifies signatures, or runs third-party code on the request path.

## 目标

高级规则生态用于把规则包导入、签名状态、来源追踪和规则测试串成一个安全闭环。第一版只支持本地 JSON 规则包，不自动联网拉取规则源。

## 规则包格式

规则包使用 JSON：

```json
{
  "id": "community-baseline",
  "name": "Community baseline",
  "version": "v1",
  "author": "LiteWaf Community",
  "license": "Apache-2.0",
  "compatibility": "litewaf-rule-package-v1",
  "defaults": {
    "enabled": false,
    "review_status": "pending-review"
  },
  "rules": []
}
```

规则字段复用现有高级规则字段：`id`、`name`、`type`、`target`、`action`、`expression`、`score`、`module`、`category`、`attack_type`、`group`、`priority`、`enabled`。

## 安全导入流程

1. 在后台“高级规则生态”粘贴本地规则包 JSON。
2. 点击预览，检查新增、变更、跳过和无效规则。
3. 关注签名状态和未签名/无效签名警告。
4. 导入后规则会作为真实规则出现在规则管理接口和页面中。
5. 对阻断型规则先运行规则测试，再绑定策略并在“发布记录”发布新版本。

预览、同步、更新预览和导出预览都不会自动启用、禁用、删除、发布或修改网关活动规则。只有导入或显式应用后，控制面规则数据才会变化；仍需发布后网关才会执行。

## 签名状态

- `verified`：本地可信 key 和校验值匹配。
- `unsigned`：规则包未提供签名，允许本地导入但发布预览会提示。
- `invalid`：签名或校验值不匹配。
- `untrusted-key`：签名 key 不在本地可信列表中。

第一版不做远程信任链、在线市场和自动更新。

## 规则测试

规则测试使用受限样例请求评估表达式，支持 path、query、headers、body 摘要和上传元数据。测试不会保存完整 body、Authorization、Cookie 或上传文件内容。

测试结果只用于上线前审查和发布预览提示。真实流量仍由发布后的网关规则执行结果决定。

## 网关兼容

网关仍只读取发布后的可执行规则。规则包 manifest、签名文件、导入历史和测试样例都不会进入网关热路径。
