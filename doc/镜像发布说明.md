# LiteWaf 镜像发布说明

本文档定义生产安装脚本消费的预构建镜像坐标和手工发布命令。正式 CI 建立前，可按本文档在构建机上发布 API、Dashboard 和 Gateway 镜像。

## 镜像坐标

生产环境通过 `deploy/.env` 控制镜像来源：

```text
LITEWAF_IMAGE_PREFIX=litewaf
LITEWAF_IMAGE_TAG=v1.0.0
```

对应镜像：

```text
${LITEWAF_IMAGE_PREFIX}/litewaf-api:${LITEWAF_IMAGE_TAG}
${LITEWAF_IMAGE_PREFIX}/litewaf-dashboard:${LITEWAF_IMAGE_TAG}
${LITEWAF_IMAGE_PREFIX}/litewaf-gateway:${LITEWAF_IMAGE_TAG}
```

生产环境应优先使用不可变版本标签，例如 `v1.0.0`。如果维护 `latest`，也必须同时发布对应的不可变版本标签，便于回滚和复现。

## 手工发布命令

在项目根目录执行：

```bash
export LITEWAF_IMAGE_PREFIX=litewaf
export LITEWAF_IMAGE_TAG=v1.0.0

docker build -t ${LITEWAF_IMAGE_PREFIX}/litewaf-api:${LITEWAF_IMAGE_TAG} .
docker build -t ${LITEWAF_IMAGE_PREFIX}/litewaf-dashboard:${LITEWAF_IMAGE_TAG} ../litewaf-dashboard
docker build -t ${LITEWAF_IMAGE_PREFIX}/litewaf-gateway:${LITEWAF_IMAGE_TAG} ../litewaf-gateway

docker push ${LITEWAF_IMAGE_PREFIX}/litewaf-api:${LITEWAF_IMAGE_TAG}
docker push ${LITEWAF_IMAGE_PREFIX}/litewaf-dashboard:${LITEWAF_IMAGE_TAG}
docker push ${LITEWAF_IMAGE_PREFIX}/litewaf-gateway:${LITEWAF_IMAGE_TAG}
```

如需同步 `latest`：

```bash
docker tag ${LITEWAF_IMAGE_PREFIX}/litewaf-api:${LITEWAF_IMAGE_TAG} ${LITEWAF_IMAGE_PREFIX}/litewaf-api:latest
docker tag ${LITEWAF_IMAGE_PREFIX}/litewaf-dashboard:${LITEWAF_IMAGE_TAG} ${LITEWAF_IMAGE_PREFIX}/litewaf-dashboard:latest
docker tag ${LITEWAF_IMAGE_PREFIX}/litewaf-gateway:${LITEWAF_IMAGE_TAG} ${LITEWAF_IMAGE_PREFIX}/litewaf-gateway:latest

docker push ${LITEWAF_IMAGE_PREFIX}/litewaf-api:latest
docker push ${LITEWAF_IMAGE_PREFIX}/litewaf-dashboard:latest
docker push ${LITEWAF_IMAGE_PREFIX}/litewaf-gateway:latest
```

## 健康检查要求

- `litewaf-api` 镜像必须包含 `/app/litewaf-api`，并支持 `/app/litewaf-api -healthcheck`。
- `litewaf-dashboard` 镜像必须包含 Nginx 配置，支持 `nginx -t`。
- `litewaf-gateway` 镜像必须包含 OpenResty 配置，支持 `/usr/local/openresty/bin/openresty -t`。

发布后建议在构建机运行：

```bash
cd deploy
cp .env.example .env
# 编辑 .env 中的 change-me 和 LITEWAF_IMAGE_TAG
./litewafctl.sh validate
docker compose --env-file .env -f docker-compose.prod.yml config
```
