# LiteWaf 规则编写指南

> 语言 / Language: [中文](规则编写指南.md) | [English](en/rule-authoring.md)

LiteWaf 规则是控制面可管理、可审计、可发布的数据。规则不会在 Dashboard 中使用 mock 数据，也不会作为隐藏网关逻辑单独存在。

## 规则字段

| 字段 | 说明 |
| --- | --- |
| `name` | 规则名称，建议稳定且可读 |
| `type` | `sqli`、`xss`、`rce`、`path-traversal`、`cc`、`bot`、`custom` |
| `target` | 检测目标 |
| `action` | `pass`、`block`、`log-only` |
| `expression` | 正则表达式，`upload_size` 使用非负整数 |
| `score` | 命中分数，范围 `0` 到 `1000` |
| `enabled` | 是否启用 |
| `module` | 用户侧防护模块，托管攻击规则为 `attack-protection` |
| `category` | 底层规则类别，托管攻击规则为 `managed` |
| `attack_type` | 托管攻击类型：`sqli`、`xss`、`rce`、`path-traversal` |
| `group` | 后台展示的托管规则组名称 |
| `priority` | 托管规则组排序和发布优先级，正整数 |

## 支持目标

| 目标 | 说明 |
| --- | --- |
| `args` | 查询参数 |
| `uri` | 原始 URI |
| `headers` | 请求头摘要 |
| `normalized_uri` | 归一化后的 URI |
| `normalized_path` | 归一化后的 path |
| `normalized_args` | 归一化后的查询参数 |
| `normalized_headers` | 归一化后的请求头摘要 |
| `body` | 请求体摘要 |
| `body_json` | JSON Body 检测 |
| `body_form` | Form Body 检测 |
| `upload_filename` | 上传文件名 |
| `upload_extension` | 上传扩展名 |
| `upload_mime` | 上传 MIME |
| `upload_size` | 上传大小 |

## 动作和评分

- `block`: 命中后阻断请求。
- `log-only`: 只记录 WAF 事件，不阻断。
- `pass`: 用于明确放行或观察场景。

策略中的 `risk_threshold` 可以让多个低分规则累计后触发阈值动作。单条 `block` 规则仍会直接阻断。

## 默认规则集

默认规则集在 [rules/default-rules.json](../rules/default-rules.json)。当前包含：

- `LiteWaf SQLi baseline`
- `LiteWaf XSS baseline`
- `LiteWaf RCE baseline`
- `LiteWaf normalized traversal baseline`

这些规则用于基础防护和验证，不代表完整托管规则库。生产启用前建议先以 `log-only` 或较高阈值观察业务流量。

默认 SQLi、XSS、RCE 和路径穿越规则会带有 `module=attack-protection`、`category=managed` 元数据，并在后台“攻击防护”模块中聚合为托管规则组。管理员可在攻击防护模块调整规则组启停、观察/阻断动作和优先级；需要维护表达式、目标字段或分数时，仍通过高级规则入口或规则 API 操作。

## 示例

SQLi 查询参数：

```json
{
  "name": "Custom SQLi union select",
  "type": "sqli",
  "target": "args",
  "action": "block",
  "expression": "(?i)union\\s+select",
  "score": 80,
  "enabled": true
}
```

JSON Body XSS 观察：

```json
{
  "name": "Observe JSON XSS",
  "type": "xss",
  "target": "body_json",
  "action": "log-only",
  "expression": "(?i)<script",
  "score": 50,
  "enabled": true
}
```

危险上传扩展名：

```json
{
  "name": "Block PHP upload",
  "type": "custom",
  "target": "upload_extension",
  "action": "block",
  "expression": "(?i)^(php|phtml|phar)$",
  "score": 90,
  "enabled": true
}
```

路径穿越托管规则示例：

```json
{
  "name": "LiteWaf normalized traversal baseline",
  "type": "path-traversal",
  "target": "normalized_path",
  "action": "block",
  "expression": "(?i)(\\.\\./|\\.\\.\\\\|/etc/passwd|/proc/self/environ)",
  "score": 70,
  "enabled": true,
  "module": "attack-protection",
  "category": "managed",
  "attack_type": "path-traversal",
  "group": "路径穿越防护",
  "priority": 130
}
```

## 限制

- 当前不执行第三方 Lua、WASM 或远程代码。
- libinjection、Hyperscan 和更大的托管规则集属于后续增强。
- 攻击防护模块当前只管理 SQLi、XSS、RCE 和路径穿越托管组，不包含访问控制、上传防护、Bot 或动态防护。
- 请求体检测需要策略显式开启，并受 Content-Type、路径前缀和大小上限约束。
- 上传检测是元数据检测，不默认读取完整文件内容。
- 动态封禁当前以网关实例本地 shared dict 为主，多实例同步是后续方向。
