# LiteWaf 规则编写指南

LiteWaf 规则是控制面可管理、可审计、可发布的数据。规则不会在 Dashboard 中使用 mock 数据，也不会作为隐藏网关逻辑单独存在。

## 规则字段

| 字段 | 说明 |
| --- | --- |
| `name` | 规则名称，建议稳定且可读 |
| `type` | `sqli`、`xss`、`rce`、`cc`、`bot`、`custom` |
| `target` | 检测目标 |
| `action` | `pass`、`block`、`log-only` |
| `expression` | 正则表达式，`upload_size` 使用非负整数 |
| `score` | 命中分数，范围 `0` 到 `1000` |
| `enabled` | 是否启用 |

## 支持目标

| 目标 | 说明 |
| --- | --- |
| `args` | 查询参数 |
| `uri` | 原始 URI |
| `headers` | 请求头摘要 |
| `normalized_uri` | 归一化后的 URI |
| `normalized_path` | 归一化后的 path |
| `normalized_args` | 归一化后的查询参数 |
| `normalized_headers` | 归一化后的请求头摘要 |
| `body` | 请求体摘要 |
| `body_json` | JSON Body 检测 |
| `body_form` | Form Body 检测 |
| `upload_filename` | 上传文件名 |
| `upload_extension` | 上传扩展名 |
| `upload_mime` | 上传 MIME |
| `upload_size` | 上传大小 |

## 动作和评分

- `block`: 命中后阻断请求。
- `log-only`: 只记录 WAF 事件，不阻断。
- `pass`: 用于明确放行或观察场景。

策略中的 `risk_threshold` 可以让多个低分规则累计后触发阈值动作。单条 `block` 规则仍会直接阻断。

## 默认规则集

默认规则集在 [rules/default-rules.json](../rules/default-rules.json)。当前包含：

- `LiteWaf SQLi baseline`
- `LiteWaf XSS baseline`
- `LiteWaf RCE baseline`
- `LiteWaf normalized traversal baseline`

这些规则用于基础防护和验证，不代表完整托管规则库。生产启用前建议先以 `log-only` 或较高阈值观察业务流量。

## 示例

SQLi 查询参数：

```json
{
  "name": "Custom SQLi union select",
  "type": "sqli",
  "target": "args",
  "action": "block",
  "expression": "(?i)union\\s+select",
  "score": 80,
  "enabled": true
}
```

JSON Body XSS 观察：

```json
{
  "name": "Observe JSON XSS",
  "type": "xss",
  "target": "body_json",
  "action": "log-only",
  "expression": "(?i)<script",
  "score": 50,
  "enabled": true
}
```

危险上传扩展名：

```json
{
  "name": "Block PHP upload",
  "type": "custom",
  "target": "upload_extension",
  "action": "block",
  "expression": "(?i)^(php|phtml|phar)$",
  "score": 90,
  "enabled": true
}
```

## 限制

- 当前不执行第三方 Lua、WASM 或远程代码。
- libinjection、Hyperscan 和更大的托管规则集属于后续增强。
- 请求体检测需要策略显式开启，并受 Content-Type、路径前缀和大小上限约束。
- 上传检测是元数据检测，不默认读取完整文件内容。
- 动态封禁当前以网关实例本地 shared dict 为主，多实例同步是后续方向。
