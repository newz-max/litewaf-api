# LiteWaf 功能需求与迭代规划

## 文档目标

本文档用于规划 LiteWaf 的功能范围和迭代节奏。项目以开源、轻便、快速部署为主题，推荐 Debian 12 minimal + Docker Compose 部署，并兼容主流 Linux + Docker Compose 环境。

功能规划原则：

- 先打通最小可用闭环，再逐步增强检测能力。
- 数据面优先保证高性能和稳定性。
- 控制面优先保证可配置、可审计、可回滚。
- 前端不使用 mock 数据，未接入接口时展示空态。
- 每个阶段都应有可运行、可验证的交付物。

## 阶段总览

| 阶段 | 主题 | 目标 |
| --- | --- | --- |
| 第 0 阶段 | 项目骨架 | 建立项目结构、基础文档、API 和前端模板 |
| 第 1 阶段 | MVP 防护闭环 | 实现站点、规则、策略、网关拦截和基础发布 |
| 第 2 阶段 | 管理能力增强 | 完善后台 CRUD、权限、审计、黑白名单、限流配置 |
| 第 3 阶段 | 日志与可观测 | 接入日志采集、查询、统计、指标和告警 |
| 第 4 阶段 | 高级防护 | 增强 SQLi、XSS、RCE、CC、Bot 等检测能力 |
| 第 5 阶段 | 生产化部署 | 完善 Debian 12 minimal 推荐部署、主流 Linux 兼容、备份、升级、回滚和安全加固 |
| 第 6 阶段 | 开源生态 | 完善文档、示例、贡献指南、规则仓库和插件机制 |

## MVP 最小验收清单

MVP 完成时至少需要满足：

| 验收项 | 状态 |
| --- | --- |
| Debian 12 minimal 或主流 Linux 上可以通过 Docker Compose 启动核心服务 | 已交付 MVP/生产 Compose 配置和生产安装脚本，待真实 Docker 环境端到端验证 |
| 后台可以创建站点、规则和策略 | 已完成 |
| 后台可以发布配置 | 已完成 |
| OpenResty 网关可以加载已发布配置 | 已完成，待端到端验证 |
| 请求可以通过 WAF 网关转发到上游 | 已完成，待端到端验证 |
| 命中基础规则时可以阻断请求 | 已完成，待端到端验证 |
| 攻击日志可以记录命中信息 | 已完成基础 WAF 事件日志和查询，待 Docker/OpenResty 环境端到端验证 |
| 发布记录可以查看 | 已完成 |
| 配置可以回滚 | 已完成 |
| 前端不展示 mock 数据 | 已完成 |
