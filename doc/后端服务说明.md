# LiteWaf API

LiteWaf API 是 LiteWaf 控制面的后端服务模板，负责防护应用管理、证书管理、规则管理、策略管理、发布回滚和日志查询等能力。

当前版本使用 Go 标准库 HTTP 服务作为基础模板，方便后续按需引入 Gin、chi、数据库驱动、Redis 客户端和鉴权模块。

## 运行

```bash
go run ./cmd/litewaf-api
```

默认监听：

```text
:8080
```

## 环境变量

| 变量 | 默认值 | 说明 |
| --- | --- | --- |
| APP_NAME | LiteWaf API | 应用名称 |
| APP_ENV | dev | 运行环境 |
| HTTP_ADDR | :8080 | HTTP 监听地址 |
| LOG_LEVEL | info | 日志级别，支持 debug/info/warn/error |
| DATABASE_URL | 空 | PostgreSQL 连接地址，空值时使用内存存储 |
| REDIS_ADDR | 空 | Redis 地址 |
| GATEWAY_CONFIG_PATH | /var/lib/litewaf/gateway/active.json | 发布后的网关配置路径 |
| AUTH_TOKEN_SECRET | dev-litewaf-change-me | 管理 API token 签名密钥，生产环境必须修改 |
| AUTH_TOKEN_TTL_MINUTES | 720 | 管理 API token 有效分钟数 |
| LITEWAF_ADMIN_USERNAME | admin | 初始化管理员账号 |
| LITEWAF_ADMIN_PASSWORD | admin123456 | 初始化管理员密码，生产环境必须修改 |
| LITEWAF_ADMIN_ROLE | admin | 初始化账号角色 |
| GATEWAY_INGESTION_TOKEN | 空 | 网关日志上报令牌，生产环境必须设置 |
| METRICS_ENABLED | false | 是否暴露 API Prometheus 指标 |
| LITEWAF_GEOIP_DB_PATH | 空 | 本地 GeoIP/IP 网段 CSV 数据库路径，空值时统计地图返回空地理数据和诊断 |
| LITEWAF_GEOIP_CACHE_SIZE | 2048 | GeoIP 解析缓存条数，设为 0 可关闭缓存 |

当 `APP_ENV=production` 时，服务会拒绝空值或已知默认弱配置，包括默认管理员密码、默认 token 密钥、默认网关上报令牌，以及包含默认数据库密码的 `DATABASE_URL`。

## GeoIP 访问地区解析

统计报表地图的访问地区由 API 使用有效客户端 IP 和 `LITEWAF_GEOIP_DB_PATH` 指向的本地 GeoIP/IP 网段库解析。请求头中的 `CF-IPCountry`、`X-Geo-*`、`X-Country-*`、`X-City`，以及访问日志上报 payload 中的 `geo_*` 字段，都不会作为地图地理来源。

当前轻量实现支持带表头的 CSV 网段库。每行可使用 `cidr`，或使用 `start_ip` / `end_ip`；可选字段包括 `country_code`、`country`、`region_code`、`region`/`province`、`city`、`district`/`county`、`longitude`、`latitude`、`source`、`version`。示例：

```csv
cidr,country_code,country,region,city,district,longitude,latitude,source,version
8.8.8.0/24,CN,中国,北京,北京,朝阳区,116.4,39.9,local-geoip,2026-06
```

部署时不要把商业或生产 GeoIP 数据提交到仓库；应通过宿主机文件或只读 volume 挂载到容器内，再设置 `LITEWAF_GEOIP_DB_PATH=/path/in/container/geoip.csv`。默认生产 Compose 已将宿主机 `./data/geoip` 只读挂载到 API 容器的 `/var/lib/litewaf/geoip`。

已安装 LiteWaf 的服务器可运行 `./litewafctl.sh geoip update` 下载 DB-IP Lite City 当月数据并转换为 LiteWaf CSV。脚本默认写入宿主机 `/opt/litewaf/data/geoip/geoip.csv`，自动设置 `LITEWAF_GEOIP_DB_PATH=/var/lib/litewaf/geoip/geoip.csv`，并重启 API 容器。后续更新继续执行同一命令即可；如需固定某个月份的数据，可使用 `LITEWAF_GEOIP_DBIP_MONTH=2026-06 ./litewafctl.sh geoip update`。

默认数据源为 DB-IP Lite City，许可证为 CC BY 4.0。页面或报表使用该数据时需要保留 DB-IP attribution。未配置、文件不可读、IP 无匹配或私有/保留地址时，系统不会生成假地区，统计报表会返回空地理数据和诊断信息。

## 接口

| 方法 | 路径 | 说明 |
| --- | --- | --- |
| GET | /healthz | 健康检查 |
| GET | /api/v1/version | 版本信息 |
| POST | /api/v1/auth/login | 管理员登录，返回访问令牌 |
| GET | /api/v1/applications | 防护应用列表 |
| POST | /api/v1/applications | 创建防护应用 |
| GET | /api/v1/certificates | 证书列表 |
| POST | /api/v1/certificates | 上传证书 |
| GET | /api/v1/rules | 规则列表 |
| POST | /api/v1/rules | 创建规则 |
| GET | /api/v1/policies | 策略列表 |
| POST | /api/v1/policies | 创建策略 |
| GET | /api/v1/attack-logs | 攻击日志 |
| GET | /api/v1/audit-logs | 审计日志 |
| GET | /api/v1/ip-access-lists | IP 黑白名单列表 |
| POST | /api/v1/ip-access-lists | 创建 IP 黑白名单 |
| GET | /api/v1/rate-limits | 限流兼容规则列表 |
| POST | /api/v1/rate-limits | 创建限流兼容规则 |
| GET | /api/v1/releases/preview | 发布前摘要和校验 |
| POST | /api/v1/releases | 发布规则 |
| POST | /api/v1/releases/{version}/rollback | 回滚到历史成功版本 |

`/api/v1/rate-limits` 仍是旧限流兼容 API，后台常规新增 URL 频率限制、登录防爆破和 API 限流应优先使用 CC 防护。旧 `/api/v1/access-lists` 黑白名单兼容 API 已移除；来源 IP/CIDR 黑白名单使用独立 `/api/v1/ip-access-lists`，路径、Header、Host 的允许、观察或阻断规则使用访问控制。

除 `/healthz`、`/api/v1/version` 和 `/api/v1/auth/login` 外，`/api/v1` 管理接口都需要 `Authorization: Bearer <token>`。角色约定：

- `admin`：可读、可写、发布、回滚、查看审计。
- `auditor`：可读配置和查看审计，不可写入。
- `readonly`：只读配置，不可写入或查看审计。

## Docker 构建

```bash
docker build -t litewaf-api .
docker run --rm -p 8080:8080 litewaf-api
```

## 部署环境

项目推荐 Debian 12 minimal 作为宿主机基线，同时兼容主流 Linux + Docker Compose 环境。后端镜像当前运行时基于 `debian:12-slim`，后续可在稳定后评估 distroless 或 scratch 运行时。
