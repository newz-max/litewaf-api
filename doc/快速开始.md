# LiteWaf 快速开始

本文档面向第一次试用 LiteWaf 的用户，目标是在 Debian 12 minimal 或其他主流 Linux + Docker Compose 环境中，用本地构建方式完成控制台登录、默认规则发布和网关拦截验证。

生产环境请优先阅读 [Debian 12 minimal 部署说明](Debian12部署说明.md)，生产安装不应在目标宿主机现场构建镜像。

## 前置条件

- Docker Engine 已安装。
- Docker Compose v2 可用。
- 当前用户可执行 Docker 命令。
- 本机端口 `18080` 和 `18081` 未被占用。

## 启动本地栈

```bash
cd deploy
docker compose up -d --build
docker compose ps
```

默认服务：

| 服务 | 地址 | 说明 |
| --- | --- | --- |
| Dashboard | `http://localhost:18080` | 后台管理页面 |
| Gateway | `http://localhost:18081` | OpenResty WAF 网关 |
| API | Dashboard `/api/` 反代 | 控制面 API |
| PostgreSQL | Compose 内部网络 | 配置、用户、日志存储 |
| Redis | Compose 内部网络 | 轻量运行状态 |
| upstream | Compose 内部网络 | 本地验证上游服务 |

默认管理员来自开发 Compose 默认环境变量：

```text
LITEWAF_ADMIN_USERNAME=admin
LITEWAF_ADMIN_PASSWORD=admin123456
```

## 创建验证配置

登录后也可以在后台完成同样操作。下面用 API 示例说明最小闭环。

```bash
curl -X POST http://localhost:18080/api/v1/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"admin123456"}'
```

将返回的 `token` 保存为变量：

```bash
export LITEWAF_TOKEN="<token>"
```

创建站点，Host 使用 `example.local`，上游指向本地验证服务：

```bash
curl -X POST http://localhost:18080/api/v1/sites \
  -H "Authorization: Bearer $LITEWAF_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Example",
    "host": "example.local",
    "upstream": "http://upstream",
    "mode": "protect",
    "enabled": true
  }'
```

查看默认规则：

```bash
curl -H "Authorization: Bearer $LITEWAF_TOKEN" \
  http://localhost:18080/api/v1/rules
```

新环境会自动写入 `LiteWaf SQLi baseline`、`LiteWaf XSS baseline`、`LiteWaf RCE baseline` 和 `LiteWaf normalized traversal baseline`。创建策略时绑定这些规则 ID：

```bash
curl -X POST http://localhost:18080/api/v1/policies \
  -H "Authorization: Bearer $LITEWAF_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Example Policy",
    "risk_threshold": 100,
    "default_action": "block",
    "normalization_enabled": true,
    "enabled": true,
    "site_ids": [1],
    "rule_ids": [1, 2, 3, 4]
  }'
```

发布配置：

```bash
curl -X POST http://localhost:18080/api/v1/releases \
  -H "Authorization: Bearer $LITEWAF_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"operator":"admin","note":"quick start validation"}'
```

## 验证网关

正常代理：

```bash
curl -i -H "Host: example.local" http://localhost:18081/echo
```

预期：HTTP `200`，响应体中的 `service` 为 `litewaf-validation-upstream`。

SQLi 拦截：

```bash
curl -i -H "Host: example.local" "http://localhost:18081/?q=union%20select"
```

预期：HTTP `403`，网关日志包含 `waf_event`。

XSS 拦截：

```bash
curl -i -H "Host: example.local" "http://localhost:18081/?q=%3Cscript%3Ealert(1)%3C/script%3E"
```

预期：HTTP `403`。

RCE-like 拦截：

```bash
curl -i -H "Host: example.local" "http://localhost:18081/?cmd=%3Bcat%20/etc/passwd"
```

预期：HTTP `403`。

更多样例见 [本地验证样例](../examples/validation/README.md)。

## 查看日志

```bash
docker compose logs -f gateway
docker compose logs -f waf-api
```

也可以在后台的攻击日志、访问日志和仪表盘页面查看真实接口返回的数据。

## 停止和清理

```bash
docker compose down
```

清空本地数据：

```bash
docker compose down -v
```
