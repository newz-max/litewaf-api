# LiteWaf 高级防护验证

本文档用于验证第 4 阶段高级防护能力：请求归一化、累计评分、请求体检测、上传元数据检测、CC 重复违规封禁和动态来源封禁。

## 配置检查

网关使用 OpenResty shared dict 保存本地限流和临时封禁状态：

```nginx
lua_shared_dict litewaf_rate_limit 10m;
lua_shared_dict litewaf_dynamic_ban 10m;
```

生产环境可根据站点数、规则数和并发规模调整 shared dict 容量。动态封禁当前是网关实例本地 best-effort 状态，不跨实例同步。

## 控制台配置

1. 在规则管理中新增或启用高级检测目标：
   - `normalized_args`：用于编码参数绕过检测。
   - `body_json`：用于 JSON 请求体检测。
   - `upload_extension`：用于危险扩展名检测。
   - `upload_size`：表达式填写字节阈值数字。
2. 在防护策略中配置：
   - 风险阈值和默认动作。
   - 请求归一化开关、解码次数和字节上限。
   - 请求体检测开关、Content-Type、路径前缀和字节上限。
   - 上传检测开关、上传字节上限和超限动作。
   - 动态封禁开关、封禁秒数、封禁分数阈值、触发次数和统计窗口。
3. 在限流配置中可选配置：
   - 违规阈值。
   - 违规窗口秒。
   - 封禁秒。
4. 发布新版本后，网关配置应包含 policy 高级防护字段。

## curl 验证样例

归一化参数检测：

```bash
curl -i -H "Host: example.local" "http://localhost:18081/?q=%2575%256e%2569%256f%256e%2520%2573%2565%256c%2565%2563%2574"
```

累计评分检测：

```bash
curl -i -H "Host: example.local" "http://localhost:18081/?a=low-risk-1&b=low-risk-2"
```

JSON Body 检测：

```bash
curl -i -X POST \
  -H "Host: example.local" \
  -H "Content-Type: application/json" \
  --data '{"q":"<script>alert(1)</script>"}' \
  http://localhost:18081/api/login
```

上传元数据检测：

```bash
curl -i -X POST \
  -H "Host: example.local" \
  -F "file=@./doc/高级防护验证.md;filename=shell.php;type=application/octet-stream" \
  http://localhost:18081/upload
```

CC 重复违规和动态封禁：

```bash
for i in $(seq 1 10); do
  curl -s -o /dev/null -w "%{http_code}\n" -H "Host: example.local" http://localhost:18081/
done
```

## 日志与 API 验证

查询攻击日志：

```bash
curl -H "Authorization: Bearer <token>" \
  "http://localhost:18080/api/v1/attack-logs?event_type=score-threshold&min_score=100"

curl -H "Authorization: Bearer <token>" \
  "http://localhost:18080/api/v1/attack-logs?advanced_target=body_json"
```

查询汇总：

```bash
curl -H "Authorization: Bearer <token>" \
  "http://localhost:18080/api/v1/observability/summary"
```

预期汇总中包含：

- `score_blocks`
- `body_detections`
- `upload_detections`
- `dynamic_bans`
