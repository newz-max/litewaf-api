# 动态防护使用指南

本文档说明 LiteWaf 第一版动态防护 / 等候室能力。当前版本保持轻量，所有网关判断都基于发布后的本地配置和 OpenResty shared dict，不在请求热路径调用控制面 API 或数据库。

## 能力范围

当前支持三类规则：

- 动态令牌：为指定路径签发和校验短期 token，适合后台、活动页等浏览器路径。
- 页面动态化：对小体积 HTML 响应注入受控标记，用于浏览器侧动态信号和后续扩展。
- 等候室：在本地网关节点按规则容量发放准入 cookie，溢出请求返回重试响应或按配置观察、阻断。

当前不包含 captcha、行为评分、设备指纹、搜索引擎识别、完整 JavaScript 混淆、任意 HTML 重写 DSL 或分布式全局队列。

## 推荐上线流程

1. 先在后台创建动态防护规则，动作选择 `log-only` 或范围较窄的路径。
2. 发布配置后观察攻击日志中的 `module=dynamic-protection` 事件。
3. 确认浏览器、移动端和 API 客户端没有误伤后，再把关键路径切换为 `block` 或 `waiting-room`。
4. 避免一开始对 `/` 全站开启阻断型动态令牌，除非已确认所有客户端都能处理 token。

## 动态令牌

动态令牌规则使用：

```text
category=dynamic-token
dynamic.mode=dynamic-token
```

关键参数：

- `token_ttl_sec`：token 有效期，建议 300 秒左右。
- `token_placement`：支持 `cookie`、`header`、`query`。
- `failure_action`：支持 `block`、`log-only`。

网关行为：

- 缺失 token 时签发新 token 并放行，日志结果为 `token-issued`。
- token 有效时放行，日志结果为 `token-passed`。
- token 过期或非法时，按 `failure_action` 观察或阻断，日志结果为 `token-failed`。
- 网关不会记录原始 token、cookie 值或签名密钥。

`cookie` 模式最适合浏览器路径。`header` 和 `query` 模式可用于能自行保存并回传 token 的客户端；首版签发会通过响应头暴露 token 名称和值，调用方需要按约定回传。

## 页面动态化

页面动态化规则使用：

```text
category=page-mutation
dynamic.mode=page-mutation
```

关键参数：

- `mutation_marker`：支持 `head-end` 或 `body-end`。
- `mutation_max_bytes`：最大可缓冲响应体，最大 1 MiB。

网关只处理 `Content-Type` 包含 `text/html` 的响应；非 HTML、超出大小上限或找不到插入标记时会安全跳过并记录 `mutation-skipped`。成功注入时记录 `mutation-applied`。网关不记录完整 HTML 响应体。

## 等候室

等候室规则使用：

```text
category=waiting-room
dynamic.mode=waiting-room
```

关键参数：

- `queue_capacity`：本地节点容量。
- `admission_ttl_sec`：准入 cookie 有效期。
- `retry_interval_sec`：溢出响应的 `Retry-After` 秒数。
- `overflow_action`：支持 `waiting-room`、`block`、`log-only`。

网关行为：

- 已持有有效准入 cookie 的请求放行，记录 `queue-admitted`。
- 容量未满时发放准入 cookie 并放行，记录 `queue-admitted`。
- 容量溢出且动作为 `waiting-room` 时返回 `503` 和 `Retry-After`，记录 `queue-queued`。
- 容量溢出且动作为 `block` 时返回 `403`，记录 `queue-blocked`。
- 容量溢出且动作为 `log-only` 时继续放行，记录 `queue-observed`。

第一版等候室状态是节点本地状态，多网关实例下不保证全局精确公平。生产多实例场景建议先用于短 TTL、低风险活动路径，后续再评估 Redis 或专用队列状态。

## 网关验证

本仓库提供动态防护 smoke 脚本：

```powershell
cd D:\Project\web_safe\codes\litewaf-gateway
.\scripts\dynamic-protection-smoke.ps1
```

脚本覆盖动态令牌签发、有效通过、过期/非法失败、观察模式、HTML 注入、`/admin` 与 `/admin2` prefix 边界、等候室溢出，以及动态防护先于托管攻击规则执行。

如果当前机器没有可用 Docker Desktop / OpenResty 环境，可先执行 JSON 和脚本语法检查，待 Linux/Docker/OpenResty 环境可用后再运行完整 smoke。

## 日志查询

动态防护事件可通过攻击日志查询：

```bash
curl -H "Authorization: Bearer <token>" "http://localhost:18080/api/v1/attack-logs?module=dynamic-protection&dynamic_result=token-failed"
```

常见 `dynamic_result`：

- `token-issued`
- `token-passed`
- `token-failed`
- `mutation-applied`
- `mutation-skipped`
- `queue-admitted`
- `queue-queued`
- `queue-blocked`
- `queue-observed`

观测汇总中的 `dynamic_protection` 按 `category|result|action|disposition` 统计。
